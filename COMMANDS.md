# HR-бот «Ченсон» — полное описание команд

Справочник для сотрудников: что делает каждая команда, кому доступна и как ей пользоваться.

---

## 👑 Команды администратора

Доступны только администраторам. Админ видит также все команды рекрутера.

---

### /add_user — добавить сотрудника

Добавляет человека в список сотрудников бота (whitelist). Пока Telegram ID
человека не добавлен этой командой, бот считает его обычным кандидатом.

**Формат:** `/add_user <telegram_id> <роль> [филиал]`

**Роли:**
- `recruiter` — рекрутер
- `branch_manager` — менеджер филиала (обязательно указать филиал)
- `chef` — шеф-повар
- `admin` — администратор

**Примеры:**
```
/add_user 123456789 recruiter
/add_user 987654321 branch_manager Чиланзар
/add_user 555555555 chef
```

Свой Telegram ID сотрудник узнаёт за 10 секунд: написать боту @userinfobot —
он пришлёт число.

---

### /users — список сотрудников

Показывает всех, кто добавлен в whitelist: ID, имя, роль и филиал.
Используйте для проверки «кто имеет доступ к боту».

---

### /bind_chat — привязать чат к филиалу или кухне

Указывает боту, в какой Telegram-чат отправлять карточки кандидатов.

**Как пользоваться:** добавьте бота в рабочий чат филиала, и **в самом этом
чате** напишите:

```
/bind_chat Чиланзар      ← карточки кандидатов Чиланзара пойдут сюда
/bind_chat kitchen       ← этот чат станет чатом «Кухня» (карточки поваров)
```

Привязку нужно сделать один раз для каждого из 4 филиалов и один раз для
чата кухни. Если чат не привязан — карточки приходят только рекрутерам
лично (ничего не теряется).

---

### /set_setting — изменить настройку

Меняет пороги автоскрининга и служебные ссылки **без программиста**.
Новое значение начинает действовать сразу.

**Формат:** `/set_setting <ключ> <значение>`

**Ключи и что они значат:**

| Ключ | Значение по умолчанию | Что регулирует |
|---|---|---|
| `waiter_age_min` / `waiter_age_max` | 18 / 30 | Возраст официантов |
| `bartender_age_min` / `bartender_age_max` | 18 / 30 | Возраст барменов |
| `tech_age_min` / `tech_age_max` | 18 / 30 | Возраст техперсонала |
| `cook_age_min` / `cook_age_max` | 25 / 50 | Возраст поваров |
| `cook_age_max_female` | 55 | Потолок возраста для женщин-поваров |
| `cook_min_exp_years` | 3 | Минимальный стаж повара (лет) |
| `drive_link` | — | Ссылка на Google Drive с медиа для публикаций |
| `kitchen_chat_id` | — | Чат «Кухня» (удобнее задавать через /bind_chat) |

**Примеры:**
```
/set_setting waiter_age_max 35        ← теперь официантов берём до 35 лет
/set_setting cook_min_exp_years 2     ← повара со стажем от 2 лет
/set_setting drive_link https://drive.google.com/...
```

---

### /settings — текущие настройки

Показывает все настройки и их текущие значения. Удобно проверить, какие
пороги скрининга сейчас действуют.

## 🧑‍💼 Команды рекрутера

Доступны рекрутерам и администраторам.

---

### /vacancies — список всех вакансий

Показывает каждую вакансию одной строкой: номер, позиция, филиал, статус
(🟢 открыта / 🔴 закрыта), прогресс закрытия «нанято X из Y».

Кандидаты видят и могут откликаться **только на открытые** вакансии.

---

### /add_vacancy — создать вакансию

Пошаговое создание вакансии. Бот спросит по очереди:

1. **Филиал** — кнопками (Куйлюк / Чиланзар / Сайрам / Сеул Мун)
2. **Позиция** — кнопками (Официант / Бармен / Повар / Техперсонал)
3. **Зарплата** — текстом, например «4–6 млн сум»
4. **График** — текстом, например «6/1, 10:00–22:00»
5. **Описание** — требования и обязанности (или «-», чтобы пропустить)
6. **Квота** — сколько человек нужно нанять, число

Вакансия сразу становится открытой и появляется у кандидатов.

---

### /close_vacancy — закрыть вакансию

Показывает список открытых вакансий кнопками — нажмите нужную, и она
закроется. Кандидаты перестанут её видеть. Набрали людей — закрывайте,
чтобы не получать лишние отклики.

---

### /set_quota — изменить квоту

Меняет план найма по вакансии. Выберите вакансию кнопкой → введите новое
число. Прогресс «нанято X из Y» пересчитается автоматически.

---

### /templates — шаблоны для публикации

Готовые тексты всех открытых вакансий для копирования в Telegram-каналы,
HeadHunter и OLX: позиция, филиал с адресом, зарплата, график, описание
и ссылка на бота. Плюс ссылка на Google Drive с фото/видео (если задана
через `/set_setting drive_link`).

Скопировали → вставили в канал → готово.

---

### /candidates — кандидаты с фильтрами

Просмотр базы кандидатов. Бот спросит фильтры по очереди:

1. **Филиал** (или «Все»)
2. **Позиция** (или «Все»)
3. **Статус** (или «Все») — например, только «Приглашён» или только «Принят»

Результат — список по 5 человек на страницу со стрелками ◀️ ▶️.
Нажмите на кандидата — откроется полная анкета с кнопкой смены статуса.

---

### /candidate — анкета по номеру

**Формат:** `/candidate 12`

Мгновенно открывает полную анкету кандидата №12: все ответы, телефон,
username, статус, причина отказа (если была), источник, дата заявки.
Номер кандидата виден в карточках и списках (например «#12»).

---

### /set_status — сменить статус кандидата

**Формат:** `/set_status 12` (по номеру) или просто `/set_status` —
бот покажет список свежих кандидатов кнопками.

Далее выберите новый статус кнопкой:

- 🆕 Новый → 📅 Приглашён → 🤝 Пришёл → 📚 Стажировка → 🎉 Принят
- ❌ Отказ работодателя — бот **обязательно** попросит причину
- 📕 Не прошёл стажировку — тоже с причиной
- 👻 Не пришёл · 🚪 Не заинтересован · ⛔️ Отказ по критериям

Каждая смена статуса записывается в историю (кто, когда, что изменил) —
на этом строится аналитика воронки.

---

### /report — отчёт за период

Пошагово: дата начала (`01.06.2026`) → дата конца → филиал (или «Все») →
позиция (или «Все»). Бот пришлёт сводку:

- **Воронка:** откликнулось → прошли скрининг → пришли на собеседование →
  вышли на стажировку → приняты (с процентами на каждом этапе)
- **Отказы по критериям** с разбивкой по причинам: возраст, опыт, язык,
  военный билет, свинина/алкоголь
- **Источники:** сколько кандидатов принёс каждый рекламный канал

Под отчётом кнопка **«📥 Скачать Excel»** — файл со ВСЕМИ анкетами периода,
включая отказы: дата, ФИО, возраст, пол, телефон, username, вакансия,
филиал, район, опыт, русский, военный билет, свинина/алкоголь, источник,
статус, причина отказа.

---

### /cancel — отмена

Прерывает любое начатое действие (заполнение вакансии, отчёта и т.д.),
если передумали или ошиблись. Работает у всех ролей.

---

## 👔 Менеджер филиала и шеф-повар

Командами пользоваться не нужно — у менеджера всё на кнопках.
Напишите боту `/start` и получите меню:

---

### ➕ Подать заявку на персонал

«Мне нужны люди» в три шага: позиция → сколько человек → комментарий по
условиям. Заявка мгновенно уходит рекрутеру, у того кнопки: создать
вакансию по заявке / уточнить детали / отклонить. О решении бот сообщит
вам автоматически.

### 📋 Открытые вакансии моего филиала

Список вакансий вашего филиала с прогрессом: «Официант — нанято 2 из 5».
(У шеф-повара этой кнопки нет.)

### 👥 Мои кандидаты

Все кандидаты вашего филиала, подтвердившие собеседование. Нажмите на
кандидата — полная анкета и кнопки:

- 🤝 **Пришёл на собеседование**
- 👻 **Не пришёл**
- 📚 **Вышел на стажировку**
- 📕 **Не прошёл стажировку** — бот попросит причину
- 🎉 **Принят**

Каждое ваше действие автоматически дублируется рекрутеру — звонить и
отчитываться не нужно.

Менеджер видит только свой филиал. Шеф-повар — поваров всех филиалов.

---

## 👤 Кандидат

Единственная команда — `/start`. Дальше всё на кнопках: «Откликнуться»
(анкета), «Список вакансий», «Условия работы». Внутри анкеты — кнопка
«⬅️ Назад» на каждом шаге и `/cancel` для отмены.

---

## ⚙️ Что бот делает сам, без команд

- **Автоскрининг** каждой анкеты по настроенным порогам — неподходящие
  получают вежливый отказ, HR не тратит на них время.
- **Карточки кандидатов** в чаты филиалов и рекрутерам — мгновенно после
  подтверждения собеседования. Повара — в чат «Кухня» и шефу, с адресом
  Куйлюка.
- **День сверки** — каждый понедельник в 10:00 рекрутеру приходит чеклист:
  открытые вакансии, кто принят за неделю, кто не прошёл стажировку
  (с причинами), на каких критериях отсеиваются кандидаты.
- **Учёт источников** — раздавайте в рекламе ссылки вида
  `t.me/<имя_бота>?start=hh`, `?start=instagram`, `?start=olx`,
  `?start=qr` — и в отчётах будет видно, какой канал приносит людей.
- **Журнал действий** — каждое действие сотрудника записывается:
  кто сменил статус, кто создал вакансию, кто изменил настройки.
